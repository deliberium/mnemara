# Contributors

## Contribution Areas

- core memory model and API design
- embedded storage backends
- retrieval and ranking evaluation
- service transport and SDKs
- privacy, deletion, and tenancy controls

## Contribution Standard

Contributors should preserve these project priorities:

- security first
- explainability over opaque magic
- measurable quality improvements
- stable compatibility policy for client integrations

## Release Scope

Features outside the first release scope should be tracked in [ROADMAP.md](ROADMAP.md) rather than merged as implied near-term commitments in contributor-facing docs.
